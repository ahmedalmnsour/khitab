
# المساهمة في خِطاب khitab

شكراً لاهتمامك بالمساهمة في **خِطاب**. المشروع جوهره لغوي: قاموس عربي للمخاطبة الجندرية في الواجهات. أثمن مساهمة هنا هي **لغوية** اقتراح صيغة عربية دقيقة أو إكمال صيغة ناقصة، ولا تحتاج خبرة تقنية بالضرورة.

يلتزم كل المشاركين بـ [قواعد السلوك](./CODE_OF_CONDUCT.md).

---

## القاعدة الأهم: اقترح النص، لا المفتاح

عند اقتراح عبارة جديدة، **تقترح النص العربي والسياق فقط، لا تقترح اسم المفتاح** (مثل `save` أو `welcomeBack`).

السبب: أسماء المفاتيح **عقد عام** تعتمد عليه كل المشاريع المستهلكة، وتغييرها لاحقاً مكلف ومُعطِّل. لذلك يسكّ المفتاحَ **صاحبُ المشروع وحده** عند اعتماد العبارة، وفق اصطلاح تسمية موحّد. هذا يحمي اتساق القاموس، ويُسهّل المساهمة على غير التقنيين.

ما يُطلب منك في الاقتراح:
- العبارة للمذكّر (مطلوبة للعبارة الجندرية).
- العبارة للمؤنّث (مطلوبة للعبارة الجندرية).
- الصيغة المحايدة (اختيارية، أو وحدها للعبارة المحايدة طبيعياً).
- **السياق**: أين ومتى تُستخدم العبارة في الواجهة.
- تأكيد أنك بحثت ولم تجد العبارة موجودة مسبقاً.

## قواعد البيانات

تنتمي كل عبارة في القاموس إلى **أحد نوعين فقط**:

### النوع الأول: عبارة جندرية

تحتوي `male` و`female` **معاً**، وقد تحتوي `neutral` اختيارياً.

```json
"save": {
  "male": "احفظ",
  "female": "احفظي"
}
```

الشروط:
1. `male` و`female` إلزاميتان معاً.
2. `male` لا تساوي `female` (نصّاً، شاملاً التشكيل).
3. `neutral` اختيارية؛ وإن وُجدت فلا تساوي `male` ولا `female`.
4. كل القيم غير فارغة.

### النوع الثاني: عبارة محايدة طبيعياً

تحتوي `neutral` **فقط**، بلا `male` و`female`. للنصوص التي لا تخاطب الشخص بجنسٍ محدّد (مبني للمجهول، صيغ مصدرية، رسائل عامة).

```json
"tryLater": {
  "neutral": "يُرجى المحاولة لاحقاً"
}
```

الشروط:
1. `neutral` إلزامية ووحيدة.
2. `male` و`female` غير موجودتين.
3. `neutral` غير فارغة.

### قواعد عامة لكلا النوعين

- **الشكل مسطّح إلزامياً**: كل المفاتيح في المستوى الأعلى مباشرة. لا تداخل (nesting) في كائنات تصنيف؛ القيمة الوحيدة داخل المفتاح هي حقول `male`/`female`/`neutral`.
- **الترتيب أبجدي إلزامياً** حسب اسم المفتاح الإنجليزي، **مفروض آلياً** عبر سكربت التحقق، لا يدوياً.
- لا تكرار في المفاتيح.
- **التشكيل إلزامي** عندما يكون الفرق الوحيد بين `male` و`female` حركةً فقط (مثل `لقد سجّلتَ` مقابل `لقد سجّلتِ`)، واختياري حين يكون الفرق نصّاً واضحاً (مثل `احفظ` مقابل `احفظي`).

## بوابة الجودة

قبل أي Pull Request، شغّل الأوامر الخمسة محلياً وتأكّد من نجاحها جميعاً:

```bash
npm run validate    # سلامة بنية وترتيب JSON
npm run lint        # جودة الكود
npm run typecheck   # سلامة الأنواع
npm run test        # الاختبارات
npm run build       # نجاح البناء
```

تكرّر منظومة CI هذه الفحوصات على GitHub، لكن تشغيلها محلياً يوفّر وقتاً ويمنع التزامات معيوبة. **لن يُدمَج أي PR لا تمرّ فحوصاته.**

## المراجعة اللغوية

لا تدخل أي عبارة القاموس قبل مراجعة لغوية. القاعدة: موافقة **2 من 3** مراجعين على الأقل (المؤلف + مراجعان متخصصان)، مع اللجوء إلى المراجع الثالث عند الخلاف. هدفنا الدقة في الفصحى، لا السرعة.

## المسار العملي للمساهمة

### اقتراح عبارة جديدة

افتح [Issue عبر قالب "اقتراح عبارة"](https://github.com/ahmedalmnsour/khitab/issues/new?template=add-phrase.md)، واملأ النص للمذكّر والمؤنّث (أو المحايدة) مع السياق. تذكّر: لا حاجة لاقتراح مفتاح — صاحب المشروع يسكّه عند الاعتماد.

### إضافة صيغة ناقصة عبر Pull Request

إن أردت إكمال صيغة ناقصة لعبارة قائمة (مثل إضافة `neutral` لعبارة جندرية شائعة):

1. اعمل Fork ثم فرعاً وصفياً (على سبيل المثال `add-neutral-save`).
2. عدّل `src/data/ar.json` بشرط الالتزام بقواعد البيانات أعلاه (بما فيها الترتيب الأبجدي والتشكيل).
3. شغّل بوابة الجودة الخمس وتأكّد من خضرتها.
4. افتح PR يشرح **النص المضاف وسياقه ومصدر صحته اللغوي**.

> ملاحظة: العبارات الناقصة لـ `neutral` مرحَّب بها بشدّة، فهي تقلّل اللجوء إلى المذكّر افتراضياً وتخدم الشمولية.
